Задача 1

Яка назва змінної вірна?

  • let user-name
  • let UserName
  • let userName

Відповідь:

При оголошенні змінної використовується стиль lowerCamelCase, як у третьому варіанті, але синтаксична помилка з'явиться лише у першому випадку. У другому випадку - код буде працювати.

Задача 2

Чи буде помилка при виконанні наступної програми?

const userName = 20
userName = "Іштван"

Відповідь:

Значення userName = 20 оголошено для константи, тому не може змінюватись, як наприклад у змінної "let"

Задача 3

Що потрапить в консоль?

let numOne = 20
let numTwo = "20"
let numSumm = numOne + numTwos
console.log(numSumm)

Відповідь:

Оператор "+" конвертує число в рядок, тому при конкатенації в консоль потрапить 2020 (тип даних string). При використанні оператора "+"(додавані) з різними типами даних, пріоритет має "рядок"(string) тому javaScript інтерпритує дії користувача як конкатенація, а не арефметичний вираз.
